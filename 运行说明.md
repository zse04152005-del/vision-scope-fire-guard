# 运行说明（Mac/Windows 通用）

## 1. 环境准备
推荐使用 Conda 新建环境：
```
conda create -n fireguard python=3.10 -y
conda activate fireguard
```

安装依赖（首次运行必做）：
```
python -m pip install ultralytics opencv-python PyQt6
```

如果提示缺少 torch，可再执行：
```
python -m pip install torch
```

## 2. 运行项目
在项目目录执行：
```
python fire_detection_system.py
```

## 3. 修改/新增摄像头
摄像头配置在 `config.json` 文件中：

- 本地 USB 摄像头：使用数字编号（0/1/2）
- 网络摄像头（RTSP）：使用 rtsp 地址

示例（一个本地摄像头）：
```json
"cameras": [
  {"id": "cam01", "name": "USB-0", "source": 0}
]
```

示例（一个 RTSP 摄像头）：
```json
"cameras": [
  {"id": "cam01", "name": "Lab-01", "source": "rtsp://user:pass@192.168.1.20:554/xxx"}
]
```

新增多个摄像头时直接往 `cameras` 数组里追加即可，修改后重启程序生效。

## 4. 常用配置说明（config.json）
- `model_path`：模型权重文件路径（默认 `best.pt`）
- `output_dir`：事件日志与截图保存目录（默认 `results`）
- `alarm.hit_threshold`：连续命中次数触发告警
- `alarm.cooldown_seconds`：同一摄像头告警冷却时间（秒）
- `alarm.conf_threshold`：最低置信度阈值
- `alarm.interval_s`：每路推理间隔（秒）
- `grid_cols`：固定网格列数（默认 3，适合 3x4 布局）

## 5. 测试（可选）
```
python -m unittest discover -s tests -v
```

## 6. 常见问题
- 没有画面：确认已点击“启动全部”，或检查摄像头编号/RTSP 地址是否正确。
- 运行报错 `No module named ultralytics`：请先安装依赖。
- Mac 上滚动条不明显：滚轮/触控板在右侧面板可上下滚动。
